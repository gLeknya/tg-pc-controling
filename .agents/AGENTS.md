# Инструкции и правила для ИИ-агентов (AGENTS.md)

Добро пожаловать в проект **tg-pc-controling**. Ниже приведены строгие правила и стандарты разработки, которым должны следовать все AI-агенты при работе с этой кодовой базой.

---

## 🔒 Безопасность и Конфиденциальность
- **Никаких жестко закодированных токенов / паролей**: Токен бота (`BOT_TOKEN`) и любые другие секреты никогда не должны находиться в коде. Все настройки должны загружаться динамически из файла `.env` (используя `os.getenv`).
- **Файлы окружения в Git**: Убедитесь, что `.env` внесен в `.gitignore` и не будет случайно отправлен в репозиторий.
- **Ограничение прав**: Любые новые административные команды или функции управления ПК должны проходить проверку через функцию `allowed(uid)`.

---

## 📁 Структура проекта
- `main.py` — основной файл запуска и логики Telegram-бота.
- `.env` — локальные секреты (токен бота). Не отслеживается Git.
- `.env.example` — шаблон настроек. Отслеживается Git.
- `.agents/mcp_config.json` — конфигурация локальных серверов MCP.
- `.agents/AGENTS.md` — текущие правила для агентов.

---

## 💻 Стандарты кодирования (Python)
- **Версия Python**: Совместимость с Python 3.10+.
- **Библиотеки**: Бот использует `python-telegram-bot` версии 20+ (асинхронный API).
  - Все хэндлеры должны быть асинхронными (`async def`).
  - Для выполнения блокирующих операций ввода-вывода (например, `os.walk`, чтение больших файлов) используйте `await asyncio.to_thread()`.
- **Разметка сообщений**: При рендеринге интерфейса в Telegram используйте `parse_mode="HTML"`. Тщательно экранируйте спецсимволы в именах файлов, чтобы избежать ошибок парсинга HTML.
- **Комментарии**: Оставляйте комментарии на русском языке. Сохраняйте существующие комментарии и структуру кодовых блоков (КОНФИГ, РЕНДЕРИНГ, ХЭНДЛЕРЫ и т.д.).

---

## ⚙️ Правила внесения изменений
1. **Проверка синтаксиса**: После редактирования `main.py` проверяйте его на синтаксические ошибки с помощью встроенного компилятора (`python -m py_compile main.py`).
2. **Создание коммитов**:
   - Пишите понятные сообщения коммитов на английском языке (например, `feat: add process monitor command`).
   - Перед созданием коммита проверяйте `git status` и `git diff`.
3. **Обновление документации**: Если вы добавляете новые команды в бота (например, `/exec`, `/sys`), обязательно обновите список поддерживаемых команд в хэндлере `/help` в `main.py` и в файле `README.md`.
