# Архитектура и структура проекта `tg-pc-controling`

В этом документе описывается архитектура приложения, роли его модулей и ключевые технические решения.

---

## 🏗 Архитектурный стиль

Приложение построена по принципу **модульного монолита**. Вся логика разделена на независимые слои (конфигурация, работа с файловой системой, модели данных, рендеринг, анимация и обработчики команд), а единой точкой входа является файл `main.py`.

## ⚙️ Ключевые технические решения

### 1. MD5 Реестр путей (Ограничение deep-link в 64 байта)
В Telegram параметр автозапуска (payload в ссылках вида `https://t.me/bot?start=payload`) строго ограничен **64 байтами**. Передавать в ссылке полный путь к файлу или папке (например, `C:\Users\Username\Documents\Projects\DeepFolder\file.txt`) невозможно.

**Решение**:
В [src/registry.py](file:///E:/PrOects/code/tg-pc-controling/src/registry.py) реализован реестр путей:
- При рендеринге пути преобразуются в MD5-хэш (берутся первые 10 символов) и записываются в глобальный словарь реестра.
- В Telegram-ссылку зашивается только этот короткий хэш-ключ (например, `cd_a1b2c3d4e5`).
- При переходе по ссылке бот извлекает оригинальный путь из реестра по хэш-ключу.

### 2. Асинхронность и неблокирующий I/O в потоках
Поскольку бот является асинхронным (использует `python-telegram-bot` v20+), любые блокирующие операции ввода-вывода (чтение дисков через `os.walk`, отправка больших файлов с диска, запуск внешних CLI-команд через `subprocess.run`) блокируют основной поток выполнения event loop, из-за чего бот перестаёт отвечать другим пользователям.

**Решение**:
Все блокирующие системные вызовы обёрнуты в стандартную асинхронную функцию:
```python
await asyncio.to_thread(blocking_function, *args)
```
Это запускает блокирующий код в отдельном системном потоке, сохраняя отзывчивость бота.

### 3. Интерактивный спиннер (Loading Indicator)
При чтении больших или сетевых директорий процесс получения списка файлов может занимать несколько секунд.

**Решение**:
- Перед началом чтения директории в [src/spinner.py](file:///E:/PrOects/code/tg-pc-controling/src/spinner.py) запускается фоновая задача `_spin_loop`.
- Задача каждые `0.8` секунды обновляет сообщение пользователя, перебирая символы спиннера (`•`, `+`, `×`, `⁕`, `⁜`, `※`) и случайное слово ожидания (`fetching`, `scrolling`...).
- По окончании загрузки задача останавливается, и сообщение обновляется финальным списком файлов.

### 4. Отсутствие жестких зависимостей (Graceful Degrade)
Команды `/sys`, `/ps` и `/kill` используют библиотеку `psutil`, которая может отсутствовать на целевой системе.

**Решение**:
Бот импортирует `psutil` локально внутри хэндлеров. Если библиотека отсутствует, бот не падает при старте, а при вызове соответствующих команд выводит информативное сообщение о необходимости установки `psutil`. Для команды `/kill` также предусмотрен fallback на системные утилиты `taskkill` (Windows) и `kill` (Linux/macOS) через subprocess.
